0

zero-knowledge

All encryption happens client-side. Servers never see your secrets in plaintext. Split-key architecture ensures no single point of failure.

1

ephemeral by design

One-time use tokens with configurable TTL. Secrets are automatically deleted after access, leaving no trace on the server.

∅

environment management

Encrypt .env files for safe version control. Decrypt on deployment with zero-knowledge guarantees using zkdotenv.

encrypt your secrets

python • zkcloud.py
from zkcloud import encrypt_env_file # encrypt .env file with zero-knowledge encrypt_env_file( '.env', '.zk.env', api_base='http://localhost:3001/api' ) # safe to commit .zk.env to version control # decryption requires both file + server access

share one-time secrets

rust • backend api
// generate token pair POST /api/tokens { "token_a": "crypto_secure_token", "token_b": "paired_token" } // one-time decrypt POST /api/decrypt { "token_a": "crypto_secure_token" } // returns token_b, then deletes both
